# Спецификация: Отрисовка (фронтенд)

## Цель
Определить, как клиент отображает игровой мир на Canvas.

## Технология
- HTML5 Canvas (2D контекст).
- Отрисовка происходит в цикле `requestAnimationFrame` (около 60 FPS), но положение объектов интерполируется между состояниями от сервера (20 FPS) для плавности.

## Что рисуем

### 1. Игроки
- Каждый игрок — геометрическая фигура, соответствующая его фракции (пока цветной круг/квадрат/ромб).
- Цвета фракций (первые четыре для MVP):
  - Фидерум: синий
  - Дин'Аслан: жёлтый
  - Фриктоз: красный
  - Апиро-Кипера: фиолетовый
- Размер: радиус `PLAYER_RADIUS = 25`.
- Оружие: серая линия, выходящая из центра игрока в направлении `angle` (длина 30 пикселей).
- Если игрок мёртв (`dead: true`) — не рисуем.
- Если игрок неуязвим (`invulnerable: true`) — рисуем с эффектом мерцания (меняем прозрачность/белый контур).

### 2. Пули
- Маленькие серые кружки радиусом 5.

### 3. Мобы
- Зелёные кружки (или другие формы) радиусом 20.

### 4. Карта и стены
- Фон карты: светло-серый.
- Стены: тёмно-серые прямоугольники (координаты и размеры получаем от сервера в начале игры).

### 5. Интерфейс (HUD)
- Список игроков с их счётом (таблица справа).
- Свой счёт и здоровье внизу.

## Камера
- Камера следует за своим игроком (центр экрана — позиция игрока).
- Мир больше экрана: `MAP_WIDTH = 9000`, `MAP_HEIGHT = 9000`, экран, например, 800x600.

## Интерполяция
- Для плавности храним предыдущее и целевое положение игроков/пуль.
- Каждый кадр сдвигаем текущую отрисовку на 10–20% к целевой.

## Псевдокод цикла отрисовки

function draw() {
clear canvas
set camera transform to follow local player
draw walls
draw mobs
draw bullets
draw players
draw HUD (без трансформации камеры)
requestAnimationFrame(draw)
}

## Данные для отрисовки
- Храним глобальное состояние, обновляемое при получении `state` от сервера.
