# Спецификация: Движение игрока

## Цель
Игрок может перемещать своего персонажа по карте с помощью клавиш WASD.

## Входные данные (от клиента)
- Флаги нажатия клавиш: `up`, `down`, `left`, `right` (булевы).
- Направление мыши: `angle` (в градусах или радианах) — не влияет на движение, но нужно для стрельбы.

## Обработка на сервере
1. Сервер получает набор данных (JSON) движения игрока.
```json 
{
  "type": "move",
  "keys": {
    "up": true,
    "down": false,
    "left": false,
    "right": true
  },
  "mouse_angle": 45.5  // угол в градусах
}
```
2. Вычисляется вектор скорости на основе нажатых клавиш:
   - Если нажаты `up` и `left`, движение по диагонали (скорость по x и y = SPEED / √2).
   - Скорость постоянная: `PLAYER_SPEED = 5` единиц за тик (тик = 50 мс).
3. Проверяется столкновение с препятствиями (коллизия) — см. `03-collision.md`.
4. Обновляются координаты игрока `(x, y)`.
5. Координаты округляются до двух знаков после запятой.

## Ограничения
- Игрок не может выйти за границы карты: `x` в пределах [0, MAP_WIDTH], `y` в пределах [0, MAP_HEIGHT].
- При одновременном нажатии противоположных клавиш (вверх+вниз) — движение блокируется по этой оси.

## Выходные данные (всем клиентам)
Сервер рассылает обновлённые координаты всех игроков (см. протокол `02-networking/01-protocol.md`).
