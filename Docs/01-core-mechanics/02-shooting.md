# Спецификация: Стрельба

## Цель
Игрок может производить выстрелы из своего оружия, нанося урон другим игрокам и мобам.

## Входные данные (от клиента)
- Событие `shoot`: отправляется при клике левой кнопкой мыши, а также периодически при её удержании (зажатии).
- Параметры выстрела:
  - `angle` (число) — направление выстрела в градусах (0–360) или радианах, соответствующее направлению мыши относительно центра игрока.
  - `player_id` — идентификатор стреляющего игрока (сервер знает, от кого пришёл пакет).

**Примечание по зажатию ЛКМ:**
- Клиент должен отправлять событие `shoot` многократно, пока кнопка мыши удерживается.
- Рекомендуемая частота отправки — не чаще, чем раз в 0.05 секунды (или один раз за игровой тик сервера, если клиент синхронизирован с ним). Это снижает нагрузку на сеть, при этом сервер сам контролирует скорострельность через cooldown.
- Серверная логика (см. ниже) уже включает проверку кулдауна, поэтому лишние выстрелы будут проигнорированы.

## Обработка на сервере
1. Сервер получает событие `shoot` от игрока.
2. Проверяется возможность выстрела:
   - У игрока есть оружие (есть всегда).
   - Скорострельность (cooldown): после каждого выстрела игрок блокируется на `SHOOT_COOLDOWN = 0.3` секунды (6 тиков при 20 тиках/сек).
3. Если выстрел разрешён, создаётся объект `Bullet`:
   - `id` — уникальный идентификатор пули.
   - `owner_id` — ID стрелка.
   - `x`, `y` — начальные координаты (равны текущим координатам стрелка).
   - `angle` — направление.
   - `speed` = `BULLET_SPEED = 10` единиц за тик.
   - `damage` = `BULLET_DAMAGE = 1` (условная единица).
   - `ttl` (time to live, как далеко может улететь пуля) = `BULLET_LIFETIME = 100` тиков (5 секунд).
4. Пуля добавляется в общий список активных пуль на сервере.
5. На каждом тике сервер обновляет позицию всех пуль: `x += cos(angle) * speed`, `y += sin(angle) * speed`. Уменьшает `ttl` на 1, удаляет пули с `ttl <= 0`.
6. При коллизии пули с игроком или мобом вызывается обработчик попадания (см. `03-collision.md`), пуля удаляется.

## Ограничения
- Пули не могут принадлежать мобам (только игроки).
- Один игрок не может стрелять чаще, чем раз в `SHOOT_COOLDOWN` (антиспам).
- Зажатие кнопки не ускоряет стрельбу, так как сервер ограничивает частоту.
- Пули не проходят сквозь стены — при столкновении со стеной пуля исчезает.

## Выходные данные (всем клиентам)
Сервер рассылает список активных пуль в каждом обновлении состояния (см. `02-networking/01-protocol.md`):
```json
{
  "bullets": [
    {"id": 1, "x": 150.2, "y": 300.7, "angle": 45},
    ...
  ]
}
