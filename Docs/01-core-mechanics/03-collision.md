# Спецификация: Коллизии (столкновения)

## Цель
Определять пересечения игроков, пуль, стен и мобов для реализации механик урона, смерти и препятствий.

## Типы коллизий

### 1. Пуля — Игрок
- Проверяется на каждом тике для каждой пули и каждого игрока (кроме владельца пули).
- Форма коллизии: окружности.
  - Игрок: радиус `PLAYER_RADIUS = 25` единиц.
  - Пуля: радиус `BULLET_RADIUS = 5` единиц.
- Если расстояние между центрами пули и игрока меньше суммы радиусов, считается попадание.
- Результат попадания:
  - Наносится урон игроку (здоровье уменьшается на `damage` пули).
  - Если здоровье игрока становится `<= 0`, игрок умирает (см. `04-respawn.md`).
  - Пуля удаляется.
  - При убийстве стрелку начисляются очки (см. `03-game-rules/01-scoring.md`).

### 2. Пуля — Стена
- Стены представлены прямоугольниками.
- Для каждой пули проверяется пересечение с каждой стеной (AABB — пуля как круг, стена как прямоугольник).
- При пересечении пуля удаляется без других эффектов.

### 3. Пуля — Моб
- Аналогично пуля-игрок, но мобы имеют радиус `MOB_RADIUS = 20` (см. `03-game-rules/02-mobs.md`).
- При попадании моб теряет здоровье, при `<= 0` исчезает, стрелку начисляются очки.

### 4. Игрок — Стена
- Игрок не может двигаться в сторону стены.

### 5. Игрок — Игрок (опционально для будущих версий)
- Пока игроки проходят друг сквозь друга (без коллизии), чтобы избежать блокировок.

## Выходные данные
- Сервер обновляет состояния объектов (здоровье, жив/мёртв) и рассылает новые данные клиентам.
- При смерти игрока клиент получает событие (например, флаг `dead` в состоянии игрока).

## Параметры (тестовые)
- `PLAYER_RADIUS = 25`
- `BULLET_RADIUS = 5`
- `MOB_RADIUS = 20`
