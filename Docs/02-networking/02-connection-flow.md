# Спецификация: Процесс подключения и комната

## Цель
Описать жизненный цикл подключения игрока к игре и управление комнатами.

## Этапы

### 1. Подключение к серверу
- Клиент устанавливает WebSocket-соединение с сервером по URL: `ws://<server-ip>:8000/ws`.
- После установки соединения клиент отправляет сообщение `join` (см. `01-protocol.md`).

### 2. Поиск или создание комнаты
- Сервер поддерживает список активных комнат (игровых сессий).
- Комната рассчитана ровно на 4 игроков.
- При получении `join` сервер ищет комнату, в которой ещё есть место (менее 4 игроков и игра не началась).
- Если такой комнаты нет, создаётся новая комната, игрок становится первым.
- Если комната найдена/создана, игрок добавляется в неё.

### 3. Ожидание заполнения
- Когда в комнате собирается 4 игрока, запускается обратный отсчёт `COUNTDOWN = 3` секунды (опционально).
- По окончании отсчёта игра начинается: игроки спавнятся (см. `04-respawn.md`), начинается рассылка состояний.

### 4. Игровой процесс
- Сервер рассылает состояние всем игрокам комнаты с частотой 20 Гц.
- Клиенты отправляют команды движения/стрельбы.

### 5. Отключение игрока
- Если игрок закрывает вкладку или теряет соединение, сервер удаляет его из комнаты.
- Если в комнате остаётся менее 4 игроков, игра может быть приостановлена или завершена (пока просто продолжаем, но новых игроков не добавляем).
- При дисконнекте всех игроков комната удаляется.

## Примечания для реализации (бэкенд)
- Хранить комнаты в памяти сервера (словарь `room_id -> Room`).
- Каждое подключение — отдельный объект, связанный с комнатой.
- Использовать asyncio для управления множеством соединений.
