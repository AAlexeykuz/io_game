# Спецификация: Сетевой протокол

## Цель
Определить форматы сообщений между клиентом (JS) и сервером (Python) через WebSocket.

## Общая структура
Все сообщения — JSON-объекты. Сервер выступает в роли источника истины: клиенты отправляют только команды, сервер рассылает состояние всем.

## Сообщения от клиента к серверу

### 1. Join (подключение к комнате)
```json
{
  "type": "join",
  "player_name": "Player1",
  "fraction": "Fiderum"  // одна из восьми фракций (пока просто строка)
}
```
### 2. Movement (движение)
```json
{
  "type": "move",
  "keys": {
    "up": true,
    "down": false,
    "left": false,
    "right": true
  },
  "mouse_angle": 45.5  // угол в градусах
}
```
### 3. Shoot (выстрел)
```json
{
  "type": "shoot"
}
```

## Сообщения от сервера к клиентам

### 1. Game state (периодическое обновление, ~20 раз в секунду)
```json
{
  "type": "state",
  "players": [
    {
      "id": 1,
      "name": "Player1",
      "fraction": "Fiderum",
      "x": 120.5,
      "y": 340.2,
      "angle": 45,
      "health": 3,
      "score": 10,
      "dead": false,
      "invulnerable": true   // для визуализации неуязвимости
    },
    // ... остальные игроки
  ],
  "bullets": [
    {
      "id": 101,
      "x": 200.1,
      "y": 150.3,
      "angle": 90
    },
    // ...
  ],
  "mobs": [
    {
      "id": 201,
      "type": "electrophage",
      "x": 500.0,
      "y": 420.0,
      "health": 1
    }
  ]
}
```

## Примечания
- Все координаты — числа с плавающей точкой.
